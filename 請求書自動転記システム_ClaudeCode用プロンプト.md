# 請求書自動転記システム｜Claude Code バイブコーディング用プロンプト

---

## メインプロンプト

```
以下の仕様に基づいて、Google Apps Script（GAS）のコードを作成してください。
ファイル構成・コード・デプロイ手順をすべて出力してください。

## システム概要

Gmailに届いた請求書PDF添付メールを定期監視し、Gemini APIでPDFを解析して
Googleスプレッドシートに自動転記するシステムをGASで実装する。

---

## 設定値（CONFIG）

コード冒頭に以下の定数オブジェクトを定義してください。

```javascript
const CONFIG = {
  SPREADSHEET_ID:    '★現在使用中のスプレッドシートID★',
  FOLDER_ID:         '★新年度ファイルの保存先GoogleドライブフォルダID★',
  GEMINI_API_KEY:    '★Gemini APIキー★',
  SHEET_SUFFIX:      '731締め',
  CLOSING_DAY:       5,
  ERROR_NOTIFY_EMAIL:'★エラー通知先メールアドレス★',
  PROCESSED_LABEL:   '請求書処理済み',
  FISCAL_YEAR_START: 4,
};
```

---

## 処理フロー

### メイン関数：checkAndProcessInvoices()

1. GmailAppで未処理メールを検索する
   - 条件：PDFファイルが添付されている & ラベル「請求書処理済み」が未付与
2. 各メールに対して以下を実行する
3. 添付PDFをBase64エンコードしてGemini APIに送信する
4. Gemini APIから以下のJSON形式でデータを抽出させる
   ```json
   {
     "name": "請求者の本名",
     "stage_name": "請求者の芸名（なければ空文字）",
     "amount": "最終合計金額（数値のみ）",
     "invoice_date": "請求日（YYYY/MM/DD形式）",
     "content": "件名または品目名（複数の場合はカンマ連結）",
     "payment_due": "支払期限（YYYY/MM/DD形式）"
   }
   ```
5. 受信日に基づいてスプレッドシートの転記先を決定する（下記ルール参照）
6. 最終行に1行追記する
7. Gmailメールに「請求書処理済み」ラベルを付与する
8. エラー発生時はシートにエラー行を記録し通知メールを送信する

---

## スプレッドシート転記先の決定ロジック

### シート名の形式
- `YYYY.M (731締め)`
- 例：`2026.6 (731締め)`

### スプレッドシート名の形式
- `YYYY年度 請求書管理`
- 例：`2026年度 請求書管理`

### 振り分けルール

```
受信月が4月（新年度）?
  └─ YES → CONFIG.FOLDER_IDのフォルダに新スプレッドシートを作成
            → 4月シートを新規作成してヘッダー行を挿入
  └─ NO  → CONFIG.SPREADSHEET_IDのスプレッドシートを使用
            → 該当月のシートが存在するか確認
               └─ なし → 新シートを作成してヘッダー行を挿入
               └─ あり → 既存シートを使用
```

### ヘッダー行（新シート作成時に1行目へ挿入）
```
到着月日 | 名前 | 芸名 | 金額 | 請求月日 | 内容 | 支払い予定
```

---

## 転記フィールドのマッピング

| 列 | 項目 | 値 |
|----|------|-----|
| A | 到着月日 | メール受信日（Date型） |
| B | 名前 | Gemini抽出の name |
| C | 芸名 | Gemini抽出の stage_name |
| D | 金額 | Gemini抽出の amount（数値） |
| E | 請求月日 | Gemini抽出の invoice_date |
| F | 内容 | Gemini抽出の content |
| G | 支払い予定 | Gemini抽出の payment_due |

---

## 締め切り超過処理

- 受信日の「日」が CONFIG.CLOSING_DAY（5日）を超えている場合：
  - G列（支払い予定）の末尾に `⚠️締切超過` を追記する
  - 超過通知メールをCONFIG.ERROR_NOTIFY_EMAILに送信する
  - 通知メールの件名：`[要確認] 締め切り超過の請求書を受信しました`
  - 通知メールの本文：受信日・送信者・金額・転記先シート名を含める

---

## エラー処理

- PDF解析失敗時：
  - 該当行のD列（金額）に `要確認` と記入して追記する
  - エラー通知メールを送信する（件名：`[エラー] 請求書の自動転記に失敗しました`）
- 必須項目が取得できない場合：
  - 取得できた項目のみ転記し、取得できなかった項目は `-` で埋める
  - エラー通知メールを送信する
- PDF以外の添付のみの場合：
  - 処理をスキップする（ラベルは付与しない）

---

## 二重処理防止

- 処理完了したメールにはGmailラベル `請求書処理済み` を付与する
- 次回のポーリング時にラベル付きメールはスキップされる
- ラベルが存在しない場合はGmailApp.createLabel()で自動作成する

---

## ファイル構成

以下の構成でコードを出力してください。

```
/
├── Code.gs          # メイン処理（checkAndProcessInvoices, triggerの設定）
├── GeminiApi.gs     # Gemini API呼び出し・JSON抽出処理
├── SheetManager.gs  # スプレッドシート・シート管理処理
├── MailNotifier.gs  # エラー通知・超過通知メール送信処理
└── appsscript.json  # GASマニフェスト（タイムゾーン等の設定）
```

---

## その他の要件

- すべての関数に日本語コメントを付けること
- エラーハンドリングは try-catch で行い、エラー内容をLogger.log()にも出力すること
- Gemini APIへのリクエストはUrlFetchApp.fetch()を使用すること
- GASの実行時間制限（6分）を考慮し、1回の実行でメールを処理しすぎないよう
  最大処理件数を10件に制限する定数 MAX_PROCESS_PER_RUN = 10 を設けること
- 完成後に「GASへの貼り付け手順」と「初回セットアップ手順」も出力すること
```
